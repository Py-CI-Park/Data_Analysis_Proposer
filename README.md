# 🧠 데이터 분석 모델 추천 시스템

## 📖 개요

데이터 분석 모델 추천 시스템은 사용자의 프로젝트 요구사항에 맞는 최적의 머신러닝, 딥러닝, 강화학습 모델을 추천하는 종합적인 웹 애플리케이션입니다. 데이터 수집부터 결과 해석까지 전체 분석 과정을 단계별로 안내하며, 각 단계에서 필요한 도구와 방법론을 제시합니다.

## ✨ 주요 기능

### 🎯 핵심 기능
- **단계별 분석 가이드**: 데이터 수집부터 결과 해석까지 8단계 체계적 프로세스
- **AI 모델 추천**: 프로젝트 특성에 맞는 최적 모델 자동 추천
- **상세한 구현 가이드**: 모델별 코드 예제와 구현 방법 제공
- **분석 리포트 생성**: 프로젝트 결과를 종합한 리포트 자동 생성
- **프로젝트 저장**: 분석 과정과 결과를 데이터베이스에 저장

### 🔧 기술적 특징
- **웹 기반 인터페이스**: 모던하고 직관적인 사용자 경험
- **RESTful API**: 확장 가능한 백엔드 아키텍처
- **SQLite 데이터베이스**: 경량화된 데이터 저장소
- **반응형 디자인**: 모바일과 데스크톱 환경 모두 지원

## 🚀 설치 및 실행

### 시스템 요구사항
- Python 3.8 이상
- 웹 브라우저 (Chrome, Firefox, Safari, Edge)
- 2GB 이상의 RAM
- 500MB 이상의 저장 공간

### 1. 프로젝트 다운로드
```bash
git clone <repository-url>
cd data-analysis-recommender
```

### 2. Python 환경 설정
```bash
# 가상환경 생성 (권장)
python -m venv venv

# 가상환경 활성화
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 의존성 설치
pip install -r requirements.txt
```

### 3. 백엔드 서버 실행
```bash
cd backend
python app.py
```

서버가 성공적으로 시작되면 다음과 같은 메시지가 표시됩니다:
```
🚀 데이터 분석 모델 추천 시스템 서버 시작...
📊 API 문서: http://localhost:5000
💾 데이터베이스 초기화 중...
```

### 4. 프론트엔드 접속
웹 브라우저에서 `frontend/index.html` 파일을 열거나, 로컬 웹 서버를 사용하여 접속합니다.

```bash
# 간단한 HTTP 서버 실행 (Python 3)
cd frontend
python -m http.server 8000

# 브라우저에서 http://localhost:8000 접속
```

### 2. 주요 페이지 및 기능

#### 🏠 홈 페이지
- 시스템 개요 및 주요 기능 소개
- 빠른 시작 버튼으로 추천 페이지 이동
- 시스템의 3가지 핵심 가치 제시

#### 📈 분석 단계 페이지
**8단계 데이터 분석 프로세스:**

1. **데이터 수집**
   - 목적: 분석 목적에 맞는 양질의 데이터 확보
   - 결과: 구조화된 데이터셋 준비
   - 도구: pandas, requests, sqlalchemy

2. **데이터 전처리**
   - 목적: 데이터 품질 향상 및 분석 준비
   - 결과: 깨끗하고 일관된 데이터
   - 도구: pandas, numpy, sklearn.preprocessing

3. **탐색적 데이터 분석(EDA)**
   - 목적: 데이터 이해 및 인사이트 발견
   - 결과: 데이터 특성 요약 및 시각화
   - 도구: matplotlib, seaborn, plotly

4. **특성 엔지니어링**
   - 목적: 예측력 높은 특성 생성
   - 결과: 최적화된 특성 세트
   - 도구: sklearn.feature_selection, pandas

5. **모델 선택**
   - 목적: 최적의 알고리즘 선택
   - 결과: 성능 좋은 모델 후보군
   - 도구: sklearn, xgboost, tensorflow

6. **모델 훈련**
   - 목적: 모델 파라미터 최적화
   - 결과: 훈련된 모델
   - 도구: 선택된 모델 라이브러리

7. **모델 평가**
   - 목적: 모델 성능 측정 및 검증
   - 결과: 성능 지표 및 평가 결과
   - 도구: sklearn.metrics

8. **결과 해석**
   - 목적: 실무진이 이해할 수 있는 인사이트 제공
   - 결과: 의사결정을 위한 결과 리포트
   - 도구: 시각화 및 리포팅 도구

#### 🤖 모델 탐색 페이지
**필터 기능:**
- 카테고리별 필터 (회귀, 분류, 클러스터링 등)
- 모델 타입별 필터 (ML, DL, RL)
- 복잡도별 필터 (초급, 중급, 고급)

**포함된 모델들:**

**머신러닝 (ML):**
- Linear Regression: 선형 회귀 분석
- Random Forest: 앙상블 기반 분류/회귀
- XGBoost: 그래디언트 부스팅
- K-Means: 클러스터링

**딥러닝 (DL):**
- CNN: 이미지 처리 특화
- LSTM: 시계열 데이터 처리
- Transformer: 자연어 처리

**강화학습 (RL):**
- Q-Learning: 가치 기반 학습
- DQN: 딥러닝 기반 Q-Learning

#### 🎯 모델 추천 페이지
**입력 정보:**
- 프로젝트 이름
- 데이터 타입 (수치형, 텍스트, 이미지, 혼합)
- 문제 유형 (분류, 회귀, 클러스터링, 시계열)
- 데이터셋 크기 (소규모, 중간, 대규모)
- 선호 복잡도 (초급, 중급, 고급)

**추천 결과:**
- 추천 점수 기반 랭킹
- 추천 이유 상세 설명
- 모델별 상세 정보 제공

### 3. 상세 기능 사용법

#### 모델 추천받기
1. **추천받기** 페이지로 이동
2. 프로젝트 정보 입력:
   ```
   프로젝트 이름: 고객 이탈 예측 모델
   데이터 타입: 수치형 데이터
   문제 유형: 분류 (Classification)
   데이터셋 크기: 중간 (10,000 - 100,000 행)
   선호 복잡도: 중급 (균형)
   ```
3. **모델 추천받기** 버튼 클릭
4. 추천 결과 확인 및 모델 상세 정보 조회

#### 분석 리포트 생성
1. 모델 추천 완료 후 **분석 리포트 생성** 버튼 클릭
2. 생성된 리포트 내용:
   - 프로젝트 요약
   - 핵심 인사이트
   - 권장 사항
   - 추천 모델 요약

#### 프로젝트 저장
1. 모델 추천 완료 후 **프로젝트 저장** 버튼 클릭
2. 저장되는 정보:
   - 프로젝트 설정
   - 추천 모델 목록
   - 분석 결과

### 4. API 문서

#### 주요 API 엔드포인트

**GET /api/analysis-steps**
- 설명: 분석 단계 목록 조회
- 응답: 8단계 분석 과정 정보

**GET /api/categories**
- 설명: 모델 카테고리 목록 조회
- 응답: 모델 분류 카테고리

**GET /api/models**
- 설명: 모델 목록 조회
- 파라미터: category_id (선택사항)
- 응답: 모델 상세 정보 배열

**POST /api/recommend-models**
- 설명: 프로젝트 요구사항에 따른 모델 추천
- 요청 본문:
  ```json
  {
    "project_name": "프로젝트명",
    "data_type": "numerical",
    "problem_type": "classification",
    "dataset_size": "medium",
    "complexity_preference": "Intermediate"
  }
  ```
- 응답: 추천 모델 목록 (추천 점수 포함)

**GET /api/model-guide/{model_id}**
- 설명: 특정 모델의 구현 가이드 조회
- 응답: 모델 구현 단계, 코드 예제, 모범 사례

**POST /api/generate-report**
- 설명: 분석 결과 리포트 생성
- 요청 본문: 프로젝트 데이터
- 응답: 종합 분석 리포트

**POST /api/save-project**
- 설명: 분석 프로젝트 저장
- 요청 본문: 전체 프로젝트 데이터
- 응답: 저장 완료 메시지

### 5. 모델 추천 알고리즘

시스템은 다음 기준으로 모델을 평가하고 추천합니다:

**점수 계산 요소:**
- 문제 유형 적합성 (가중치: 3점)
- 데이터 타입 적합성 (가중치: 2-3점)
- 데이터셋 크기 적합성 (가중치: 1-2점)
- 복잡도 일치성 (가중치: 1점)

**추천 로직:**
1. 각 모델에 대해 점수 계산
2. 점수가 0보다 큰 모델만 선별
3. 점수 순으로 정렬하여 상위 5개 모델 추천
4. 각 모델에 대한 추천 이유 제공

## 🐛 문제 해결

### 일반적인 문제들

**서버가 시작되지 않는 경우:**
- Python 버전 확인 (3.8 이상 필요)
- 의존성 설치 확인: `pip install -r requirements.txt`
- 포트 5000이 사용 중인지 확인

**프론트엔드에서 API 호출 실패:**
- 백엔드 서버 실행 상태 확인
- CORS 설정 확인
- 브라우저 개발자 도구에서 네트워크 탭 확인

**데이터베이스 오류:**
- database.db 파일 권한 확인
- SQLite 설치 확인
- 데이터베이스 재초기화: 파일 삭제 후 서버 재시작

**모델 추천이 작동하지 않는 경우:**
- 모든 필수 필드 입력 확인
- 서버 로그에서 오류 메시지 확인
- API 엔드포인트 응답 상태 확인

### 로그 확인
서버 실행 시 콘솔에 표시되는 로그를 통해 문제를 진단할 수 있습니다:
```
INFO: 데이터베이스 초기화 완료
INFO: 모델 데이터 로드 완료
ERROR: API 호출 실패 - 잘못된 요청 형식
```
